Pitchbend Filter by Note Channel

This JSFX script filters the Pitchbend messages by the last Note On's channel, while leaving all other messages intact. That is, if a Note On is encountered, it will filter out any pitchbends with a channel that does not match the channel of the note, until the next Note On is received which may change this channel. Keep in mind that even if these pitchbends may be filtered out, their values are still remembered, so that if a new Note On happens that uses another channel's pitchbend, its value will be restored to the exact last value it had (even though it wasn't sent before).

Download it here

The main purpose of this script arised from the Key Tuner. This is because the Key Tuner splits notes on different channels with different pitchbends, so we want a way to filter out the respective pitchbends for a given note's channel. If you have a note splitter available but cannot split the Pitchbends properly to different paths/synths, then just send the entire pitchbend bus to the different destinations, and use this script as the first thing in each of them to filter out pitchbends that are unwanted, by using the notes and their channels (that were split).

The reasoning is simple: most plugins ignore the input channel and thus it is impossible to play them polyphonically with the Key Tuner by having different pitchbends for each note in the polyphony. However, we can duplicate them (in any way you can think of), so that each note gets a different copy of the synth. Some hosts even allow you to do this automatically, such as Bidule via its Polyphonic Adapter, while others only allow simple note splitting or cannot do it at all (and you must do it manually).

Hopefully, the workings of this script should now be clear: it simply looks at the channel of the Note On received, and accepts only Pitchbends from that channel from this point on. Now the synth will only receive pitchbends that match its note. Please do not send this script two notes at the same time, it will only take into account the last note received (which can be arbitrary). It was not made to handle such case, because if you use multiple notes at once, you need multiple paths to split them to! (and several instances of this script on each). And don't forget that it does keep track of all pitchbends received, even those which are not on the "active channel" for the given path. This ensures the pitchbend will always be what it should be even if the new Note On is a different channel (which is likely to be).

Lastly, if you can just split the respective channels along with notes & pitchbends and route them to different synths, then do it that way, you won't have to use this script at all. This depends on your usage and host's capabilities, but also on certain extra functionality in the host, so in case you need it, this script is here…
