Pitchbend Adder

Pitchbend Adder

Simple JSFX script used to add an audio signal to Pitchbend events on the channels specified. The audio signal should be "normalized" between -1 and 1, representing the pitchbend range, which gets added to the pitchbend messages already present in the MIDI bus (but also creating new pitchbend messages if needed). So be careful! You can actually get out of range quite easily (it will get capped to max/min values in such a case). Other MIDI events are passed through. The input audio signal that gets added to the Pitchbend is mono, and it is added to all the channels specified. If you use it in a stereo configuration, it will probably be the left channel.

Download it here

There are only two parameters:

  • Channels: Specifies how many channels to affect, others are passed through (unchanged). It always starts with Channel 1, if you need other configurations, remap and reroute them before sending them to this script.
  • Message Frequency (Hz): This is simply the frequency at which to output Pitchbend messages, to limit them (i.e the frequency at which they are created from the audio signal). Do not set it too high! Some hosts have a limited amount of MIDI events per buffer (or per second) available. Others have performance problems with too many MIDI events. Just don't go overboard and keep in mind that each channel has messages of its own, not just one channel (and they all count to this limit together). One thing to remember is that regardless of this parameter, there will always be a pitchbend sent at the times occupied by the input pitchbends with the correct values (unless it's redundant). This is to remain consistent if a specific pitchbend is absolutely needed at a specific time to sound "just right", so don't worry about it, you can set the parameter low without sacrificing "accuracy" for important event timings. It only sacrifices accuracy in the actual added (and created) signal…

As you can imagine, the purpose of this little script is to be able to manipulate Pitchbend from audio, while adding it on top of other Pitchbend messages already present. If there are no such messages, then it will simply create Pitchbend messages out of the audio signal (because it is added to zero), while its precision is not limited to buffer chunks (unlike ReaControlMIDI), so it is superior even if you use it only to create pitchbends out of audio signals.

Pitchbend Adder (Active Only)

This is the same as the Pitchbend Adder, except that it only outputs Pitchbend messages on active notes in a channel. It does this by tracking how many notes are still open on which channels, and as long as there's more than one, it keeps outputting pitchbends. When there's no note played on a given channel, it won't output pitchbends on that channel. This is useful to save on performance or MIDI limits of your host, since silent channels probably won't require pitchbends to begin with.

Don't use it if you know that a silent channel can be affected by Pitchbend (or has to be), though. For example, synth tails or continuous sound generators, will not like it.

Pitchbend Adder Omni

A variant that's made purely for performance and simplicity. It is the same as the Pitchbend Adder, but it ignores the input channel and always outputs pitchbends on channel 1. This is basically sort of like a "one channel" version of Pitchbend Adder, but be sure that the input MIDI has only channel 1, else it might mess things up. If you use only one channel, though, then go ahead and choose it.

Pitchbend Adder Omni (Active Only)

This is a combination of both variants. It ignores the input channel like Pitchbend Adder Omni, and uses only one channel always, but it does not output pitchbends if no notes are active. It has the advantages of both variants in case you needed both.
