Pitchbend Randomizer

This JSFX script is similar to the Pitchbend Adder, except that it does not use an audio signal to add to the pitchbends on given channels. Instead, it generates smooth random signals in the same way as the Random Smooth Signal script, but it only does it for pitchbends, and then adds them. The difference is that it also generates a different signal for each channel — they will not be the same.

Download it here

The parameters are the same as the Pitchbend Adder, with the extra ones affecting the random generated signal in the same way as Random Smooth Signal:

  • Channels: Specifies how many channels to affect, others are passed through (unchanged). It always starts with Channel 1, if you need other configurations, remap and reroute them before sending them to this script.
  • Message Frequency (Hz): This is simply the frequency at which to output Pitchbend messages, to limit them (i.e the frequency at which they are created). Do not set it too high! Some hosts have a limited amount of MIDI events per buffer (or per second) available. Others have performance problems with too many MIDI events. Just don't go overboard and keep in mind that each channel has messages of its own, not just one channel (and they all count to this limit together). One thing to remember is that regardless of this parameter, there will always be a pitchbend sent at the times occupied by the input pitchbends with the correct values (unless it's redundant). This is to remain consistent if a specific pitchbend is absolutely needed at a specific time to sound "just right", so don't worry about it, you can set the parameter low without sacrificing "accuracy" for important event timings. It only sacrifices accuracy in the actual added (and created) random signal…
  • Lower Value: The lowest value that the random generated pitchbend signal can have, so it is the lower limit of the internal random generator. Note that it is in terms of normalized pitchbend values, which goes from -1…1 and which is added on top of other pitchbend messages that are sent to this script.
  • Upper Value: The highest value that the random generated pitchbend signal can have. Like Lower Value, it is in terms of normalized pitchbend values, and with these two, you can define the range of the random pitchbend signal. Of course, because it gets added to input pitchbend that is sent to the script, the actual pitchbend can be outside this range, so please take care of it, else it will clip at the extremes (-1 or 1).
  • Change Duration (ms): This defines the duration of one random value, in milliseconds. The longer the duration, the slower the curve will appear. Short durations will have more changes. For example, a duration of 100ms means a new random value will be generated after each 100ms, so that means 10 times per second. Please note that the Message Frequency will be capped depending on this parameter. It makes no sense to have a lower frequency than the duration allows. For example, a Change Duration of 100ms will cap the Message Frequency to 10Hz minimum — there is no point for there to be less messages, as some changes would not even be "visible", so you can't go lower than 10Hz in this case. (because 100ms is one tenth of a second)

Please see Random Smooth Signal for more information on how the randomized signal is generated and how it looks like. Even though it is similar to it, this script operates fully in the MIDI domain, no audio signals are present whatsoever, not even internally. You don't have to worry about it, but the code can be quite messy because of this fact.

Once again, just like the Pitchbend Adder, it will always output a pitchbend where an original input event is timed at, or at the start of a new note. This is to keep it always in sync regardless of the Message Frequency. However, the curve is randomly reset whenever there is a new note from an inactive channel, so there will be a discontinuity there. This is to give each brand new note that activates a given channel a different random variation and not carry over from a former note that already ended. Keeps it more natural and randomized. If you don't want this behavior, filter the notes before sending MIDI to this script, and merge them later with the pitchbends on the MIDI bus (after the script).

The purpose of this script is to add some subtle (or not so subtle) variations to pitchbends, or use them as a control signal for some other parameter by linking them. You can even go crazy with it, to simulate some insane pitch effects. You could also make it by combining the Pitchbend Adder with the Random Smooth Signal as input, but that will have the same random signal on all channels (this one generates different randomized signals on each channel!), and it is also much worse in performance since it deals with audio…

Pitchbend Randomizer (Active Only)

This is the same as the Pitchbend Randomizer, except that it only outputs Pitchbend messages on active notes in a channel. It does this by tracking how many notes are still open on which channels, and as long as there's more than one, it keeps outputting pitchbends. When there's no note played on a given channel, it won't output pitchbends on that channel. This is useful to save on performance or MIDI limits of your host, since silent channels probably won't require pitchbends to begin with.

Don't use it if you know that a silent channel can be affected by Pitchbend (or has to be), though. For example, synth tails or continuous sound generators, will not like it.

Also please remember that both this variant and the Pitchbend Randomizer will reset the random signal shape when a note happens to "activate" a channel, because it assumes no sound was prior to that, so this way it is more random overall. But in this case, filtering the notes before sending them makes no sense, since it won't output anything. (because it requires notes to activate channels and create pitchbends)

Pitchbend Randomizer Omni

A variant that's made purely for performance and simplicity. It is the same as the Pitchbend Randomizer, but it ignores the input channel and always outputs pitchbends on channel 1. This is basically sort of like a "one channel" version of Pitchbend Randomizer, but be sure that the input MIDI has only channel 1, else it might mess things up. If you use only one channel, though, then go ahead and choose it.

Pitchbend Randomizer Omni (Active Only)

This is a combination of both variants. It ignores the input channel like Pitchbend Randomizer Omni, and uses only one channel always, but it does not output pitchbends if no notes are active. It has the advantages of both variants in case you needed both.
